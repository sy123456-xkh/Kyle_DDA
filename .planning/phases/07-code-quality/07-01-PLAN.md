# Phase 7 Plan 01: 代码质量与规范配置

## 目标

为后端（Python）和前端（TypeScript）配置 linter 和 formatter，添加运行脚本，并修复存量违规。

## 当前状态

**后端：**
- mypy 已配置（mypy.ini），但不在 black/ruff 覆盖范围
- 无 pyproject.toml，无 black/ruff/isort 配置
- ~20 处行长超过 88 字符违规

**前端：**
- 无 ESLint 配置（.eslintrc 不存在）
- 无 Prettier 配置
- package.json 无 lint/format 脚本
- TypeScript strict 已开启，编译通过

---

## Task 1: 后端 — 配置 pyproject.toml + 安装工具

**创建** `backend/pyproject.toml`
**删除** `backend/mypy.ini`（mypy 配置迁移到 pyproject.toml，避免冲突）
**修改** `backend/requirements.txt`

pyproject.toml 配置（包含 mypy 配置，取代 mypy.ini）：
```toml
[tool.black]
line-length = 88
target-version = ["py311"]
exclude = '''
/(
    \.venv
  | data
)/
'''

[tool.ruff]
line-length = 88
target-version = "py311"
exclude = [".venv", "data"]

[tool.ruff.lint]
select = ["E", "F", "W", "I"]  # pycodestyle errors/warnings + pyflakes + isort
ignore = ["E501"]               # 长行由 black 控制，ruff 不重复报

[tool.mypy]
python_version = "3.11"
warn_return_any = true
warn_unused_configs = true
disallow_untyped_defs = true
exclude = ["^\\.venv/", "^data/"]
```

requirements.txt 新增：
```
black==24.10.0
ruff==0.8.6
```

验证：
- `cd backend && pip install black ruff` 安装成功
- `cd backend && python -m black --check app/` 输出（可能有违规，Task 2 修复）
- `cd backend && python -m ruff check app/` 输出

完成标准：
- pyproject.toml 存在且包含 black/ruff/mypy 配置
- requirements.txt 含 black 和 ruff

---

## Task 2: 后端 — 运行 black 格式化修复存量违规

**修改** `backend/app/*.py` 和 `backend/app/services/*.py`（由 black 自动修改）

实现：
- 在 backend/ 目录运行 `python -m black app/`（格式化所有 Python 文件）
- 运行 `python -m ruff check app/ --fix`（自动修复 isort 等可自动修复的问题）
- 对无法自动修复的 ruff 警告，手动评估是否需要处理（预计极少）
- 为关键函数添加 docstring（一行说明即可）：
  - `backend/app/main.py`：所有 endpoint 函数
  - `backend/app/services/query_service.py`：所有公开函数（私有 _ 前缀函数已有注释可跳过）
  - `backend/app/services/dataset_service.py`：所有公开函数
  - `backend/app/schemas.py`：所有 Pydantic 模型类
- 不修改逻辑，只做格式调整和文档补充

验证：
- `python -m black --check app/` 输出 "All done! ✨ 🍰 ✨" 无违规
- `python -m ruff check app/` 输出无错误
- `python -m mypy app/` 仍然通过（无新类型错误）

完成标准：
- black check 通过
- ruff check 通过
- mypy 通过（不得引入新错误）

## Task 3: 前端 — 配置 ESLint + Prettier

**创建** `frontend/.eslintrc.json`
**创建** `frontend/.prettierrc`
**修改** `frontend/package.json`

.eslintrc.json：
```json
{
  "extends": [
    "next/core-web-vitals",
    "next/typescript"
  ],
  "rules": {
    "no-unused-vars": "off",
    "@typescript-eslint/no-unused-vars": ["warn", { "argsIgnorePattern": "^_" }],
    "@typescript-eslint/no-explicit-any": "warn",
    "react/display-name": "off"
  }
}
```

.prettierrc：
```json
{
  "semi": false,
  "singleQuote": false,
  "tabWidth": 2,
  "trailingComma": "es5",
  "printWidth": 100,
  "plugins": []
}
```

package.json 新增 devDependencies：
```json
"eslint": "^9",
"eslint-config-next": "^15.0.0",
"prettier": "^3"
```

package.json 新增 scripts：
```json
"lint": "next lint",
"format": "prettier --write .",
"format:check": "prettier --check ."
```

验证：
- `cd frontend && npm install` 安装成功
- `cd frontend && npm run lint` 无错误（可能有 warning）
- `cd frontend && npm run format:check` 检查格式

完成标准：
- .eslintrc.json 和 .prettierrc 存在
- `npm run lint` 可运行且无 error（warning 可接受）
- package.json 含 lint/format 脚本
