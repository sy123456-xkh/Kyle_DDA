# Phase 5 Plan 02: UX 集成

## 目标

将 Plan 01 创建的基础组件集成到 UploadZone、workspace、data-hub 页面，完成 Phase 5 全部验收标准。

## 前置条件

Plan 01 已完成：Toast、ErrorBoundary、Skeleton 组件可用，ToastProvider 已注入 layout.tsx。

## 说明

- 拖拽上传已在 UploadZone.tsx 中实现（REQ-FUNC-001 已满足），Task 1 仅做 UX 增强
- 上传进度条为模拟进度（MVP 阶段，fetch API 不支持上传进度回调），这是已知限制
- workspace/page.tsx 当前使用 raw fetch 而非 api client，本计划不迁移（避免大范围重构），仅在现有代码基础上集成 UX 组件

---

## Task 1: 增强 UploadZone 组件

**修改** `frontend/app/components/UploadZone.tsx`

实现：
- 文件大小前端校验：>50MB 显示 Toast 警告并拒绝
- 上传进度条：模拟进度动画（0→90% 渐进，完成后跳 100%）
- 上传状态：idle → uploading → success/error
- uploading 状态下禁用拖拽和点击（防重复提交）
- 拖拽悬停时：图标从上传变为放下，文字从"拖拽或点击上传"变为"松开以上传"
- 上传成功/失败通过 onSuccess/onError 回调通知父组件（父组件决定是否 Toast）

验证：
- `npx tsc --noEmit` 无类型错误
- 拖拽文件到区域，视觉反馈变化
- 上传中进度条可见，按钮禁用
- >50MB 文件被拒绝

完成标准：
- 上传全流程有视觉反馈（idle→拖拽→上传中→完成/失败）
- 大文件前端拦截

## Task 2: workspace 页面 UX 集成

**修改** `frontend/app/workspace/page.tsx`

实现：
- 用 ErrorBoundary 包裹三栏主体区域（左/中/右各一个）
- 数据 profile 加载时：显示 SkeletonCard 替代当前 spinner
- 查询结果加载时：显示 SkeletonTable 替代当前 bounce dots
- 上传成功后：Toast success 提示 "数据集 {id} 导入成功"
- 上传失败：Toast error 提示错误信息
- 查询失败：Toast error 提示，保留 chat 中的错误消息显示
- 保留现有 raw fetch 调用不变，仅在 .then/.catch 中添加 Toast 调用

验证：
- `npx tsc --noEmit` 无类型错误
- 上传 CSV → 看到 SkeletonCard → profile 显示 → Toast 成功
- 输入查询 → 看到 SkeletonTable → 结果显示
- 模拟错误 → ErrorBoundary fallback 显示 → 点击重试恢复

完成标准：
- 所有异步操作有 loading → success/error 反馈链路
- 无白屏崩溃风险（ErrorBoundary 兜底）

## Task 3: data-hub 页面 UX 集成

**修改** `frontend/app/data-hub/page.tsx`

实现：
- 用 ErrorBoundary 包裹页面主体
- Profile 加载时显示 SkeletonCard
- 上传成功/失败使用 Toast 通知
- 移除现有红色 error banner，改用 Toast error
- 保持与 workspace 一致的交互模式

验证：
- `npx tsc --noEmit` 无类型错误
- 上传 → Skeleton → Profile → Toast 成功
- 错误时 Toast 弹出，无 inline 红色 banner

完成标准：
- data-hub 与 workspace 交互体验一致
- 错误处理统一使用 Toast
