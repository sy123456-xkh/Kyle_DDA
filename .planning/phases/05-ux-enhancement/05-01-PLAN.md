# Phase 5 Plan 01: UX 基础组件

## 目标

创建 Toast 通知、Error Boundary、Skeleton 三个基础 UX 组件，并将 ToastProvider 注入 layout.tsx。

## 当前状态

- 无 Toast 系统，错误仅 inline 红色文本显示
- 无 Error Boundary，渲染错误导致白屏
- 无 Skeleton，加载时仅有 spinner
- layout.tsx 当前无任何 Provider 包裹

---

## Task 1: Toast 通知系统 + layout.tsx 注入

**创建** `frontend/app/components/Toast.tsx`
**修改** `frontend/app/layout.tsx`

实现：
- ToastProvider（Context）+ useToast hook
- show(message, type) 方法，type: success | error | info
- 自动 3 秒消失，可手动关闭（X 按钮）
- 右上角固定定位，多条堆叠
- amber 色系：success 绿色，error 红色，info amber
- 在 layout.tsx 的 `{children}` 外层包裹 `<ToastProvider>`

验证：
- `npx tsc --noEmit` 无类型错误
- 浏览器中手动调用 useToast().show('test', 'success') 可见通知弹出

完成标准：
- Toast.tsx 导出 ToastProvider 和 useToast
- layout.tsx 包含 ToastProvider
- 三种类型通知视觉可区分

---

## Task 2: Error Boundary 组件

**创建** `frontend/app/components/ErrorBoundary.tsx`

实现：
- React Class Component（getDerivedStateFromError + componentDidCatch）
- Props: children, fallback?(可选自定义 fallback)
- 默认 fallback UI：错误图标 + "出了点问题" + 错误信息 + "重试" 按钮
- 重试按钮 setState({ hasError: false }) 触发重新渲染
- 保持 amber 主题风格，圆角卡片样式

验证：
- `npx tsc --noEmit` 无类型错误
- 创建一个故意抛错的测试组件，确认 fallback 显示
- 点击重试后组件重新渲染

完成标准：
- ErrorBoundary 可包裹任意子组件
- 渲染错误显示友好 UI 而非白屏
- 重试按钮可恢复正常渲染

---

## Task 3: Skeleton 加载占位组件

**创建** `frontend/app/components/Skeleton.tsx`

实现：
- SkeletonLine：单行占位，props: width?（默认 100%）、height?（默认 16px）
- SkeletonCard：卡片占位（模拟 DataProfile 三卡片布局），3 个灰色矩形
- SkeletonTable：表格占位（5 行 x 4 列灰色条），模拟查询结果
- 统一使用 Tailwind `animate-pulse` + `bg-gray-200 rounded`
- 导出三个命名组件

验证：
- `npx tsc --noEmit` 无类型错误
- 各组件独立渲染正常，pulse 动画可见

完成标准：
- 三种 Skeleton 变体可独立 import 使用
- 动画流畅，视觉与 amber 主题不冲突
