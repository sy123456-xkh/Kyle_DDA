# Phase 8 Plan 01: 后端测试（pytest）

## 目标

为后端核心功能编写 pytest 测试：SQL 护栏、CSV 上传、数据 profiling、聊天查询。

## 当前状态

- 无任何测试文件，无 tests/ 目录
- backend/requirements.txt 中无 pytest、httpx、pytest-cov

---

## Task 1: 测试基础设施

**创建** `backend/tests/__init__.py`
**创建** `backend/tests/conftest.py`
**修改** `backend/requirements.txt`

requirements.txt 新增：
```
pytest==8.3.4
pytest-cov==6.0.0
httpx==0.28.1
```

conftest.py 内容：
- `tmp_data_dir` fixture：每次测试使用 tmpdir 作为 DATA_DIR，隔离文件
- `test_client` fixture：使用 FastAPI TestClient（httpx），每次测试重置 DuckDB 连接
- `sample_csv` fixture：返回一个简单 CSV bytes（3列：date/product/revenue，5行数据）
- `uploaded_dataset` fixture：调用上传 API，返回 dataset_id，供其他测试复用

monkeypatch settings.DATA_DIR 到 tmp 目录，确保测试不写入 backend/data/

验证：
- `cd backend && pytest --collect-only` 正常收集（无 import error）

完成标准：
- conftest.py 可正常 import
- fixtures 可被其他测试模块使用

---

## Task 2: SQL 护栏 + 查询测试

**创建** `backend/tests/test_guardrails.py`
**创建** `backend/tests/test_query.py`

test_guardrails.py（单元测试，直接测 _guard_sql 函数）：
```python
# 需要从 app.services.query_service import _guard_sql
# 测试用例：
# - SELECT 语句通过
# - INSERT/UPDATE/DELETE/DROP/ALTER 被拒绝（各一个用例）
# - 无 LIMIT 时自动补 LIMIT 5000
# - 已有 LIMIT 时不再补
# - 超长 SQL（>5000字符）被拒绝
```

test_query.py（集成测试，通过 test_client 调用 API）：
```python
# POST /chat/query
# - 问"行数"返回 COUNT 查询
# - 普通问题返回 SELECT * LIMIT 50
# - 不存在的 dataset_id 返回 404/400
# - 包含 DROP 的 question 通过护栏（question 本身不执行，SQL 由规则生成）
```

验证：
- `cd backend && pytest tests/test_guardrails.py tests/test_query.py -v`

完成标准：
- 护栏单元测试 7 个用例全部通过
- 查询集成测试 4 个用例全部通过

## Task 3: 上传 + Profiling 测试 + 覆盖率

**创建** `backend/tests/test_upload_profile.py`

test_upload_profile.py：
```python
# POST /datasets/upload
# - 上传合法 CSV → 返回 dataset_id，格式为字母数字下划线
# - 上传非 CSV 文件（.txt）→ 返回 400
# - 上传空文件 → 返回 400 或可处理

# GET /datasets/{id}/profile
# - 有效 dataset_id → 返回 row_count/columns/missing_rate/sample_values
# - 无效 dataset_id → 返回 404/400

# profile 内容验证：
# - row_count == sample CSV 的行数（5）
# - columns 包含预期字段名
# - missing_rate 每项 0~1 之间
```

运行覆盖率：
- `cd backend && pytest --cov=app --cov-report=term-missing`
- 目标：≥ 60%

验证：
- 所有测试通过
- 覆盖率报告输出，backend coverage ≥ 60%

完成标准：
- 上传/profile 测试全部通过
- pytest --cov 显示覆盖率 ≥ 60%
