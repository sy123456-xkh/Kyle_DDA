# Phase 6 Plan 01: 饼图支持 + 图表组件抽取

## 目标

新增饼图渲染能力，同时将 workspace 中 ~100 行内联 ECharts 代码抽取为独立 ChartView 组件，为 Plan 02 的配置面板做准备。

## 当前状态

- workspace/page.tsx 内联 ECharts useEffect（lines 120-225），支持 line/bar
- 后端 ChartSpec.type 支持 "line" | "bar" | "table"
- Cross playbook 返回 type="table"，无可视化

---

## Task 1: 后端新增 pie 支持

**修改** `backend/app/schemas.py`
**修改** `backend/app/services/query_service.py`

实现：
- ChartSpec.type 注释更新为 "line" | "bar" | "pie" | "table"
- Cross playbook 改为返回 type="pie"（而非 "table"）
- Cross playbook SQL 修改：去掉 dt 时间分组，仅按 dim_col 聚合
  - 改为：`SELECT "{dc}", {AGG}("{mc}") AS value FROM {view} GROUP BY "{dc}" ORDER BY value DESC LIMIT 10`
  - 原因：饼图需要每个维度一个值，多行同维度会导致数据错误
- Cross playbook 的 ChartSpec 设置 x=dim_col, y="value"

验证：
- 后端启动无报错
- POST /playbook 发送 cross 请求，返回 chart.type="pie"
- 返回数据每个 dim_col 值只出现一次

完成标准：
- ChartSpec 支持 pie 类型
- Cross playbook 返回 pie chart spec，数据按维度聚合

---

## Task 2: 创建 ChartView 组件

**创建** `frontend/app/components/ChartView.tsx`

实现：
- Props 定义扩展的 overrides 类型：
  ```typescript
  interface ChartOverrides extends Partial<ChartSpec> {
    colorTheme?: 'indigo' | 'amber' | 'emerald' | 'rose'
    legendPosition?: 'top' | 'bottom' | 'left' | 'right'
  }
  ```
- Props: `{ spec: ChartSpec, overrides?: ChartOverrides, className?: string }`
- 导出 ChartOverrides 类型（供 Plan 02 ChartConfig 使用）
- 定义颜色主题常量 COLOR_THEMES：4 个主题各 5 色序列（indigo/amber/emerald/rose）
- 导出 COLOR_THEMES（供 ChartConfig 共享）
- overrides 允许前端覆盖 spec 中的 type/x/y（为配置面板准备）
- 合并逻辑：finalSpec = { ...spec, ...overrides }（overrides 中非 undefined 的字段覆盖）
- 动态 import echarts（与现有模式一致）
- 支持三种图表类型：
  - **line**: 复用现有 workspace 中的 line option（smooth 曲线 + 渐变）
  - **bar**: 复用现有 workspace 中的 bar option（渐变填充 + 自动旋转）
  - **pie**: 新增
    - tooltip: trigger "item"，显示名称+值+百分比
    - series: type "pie"，radius ["40%", "70%"]（环形图更现代）
    - label: 显示名称+百分比
    - 颜色序列：indigo/purple/blue/cyan/teal 渐变色板
    - emphasis: 放大效果
- type="table" 或无数据时不渲染（返回 null）
- 响应式 resize + dispose 清理
- 暗色主题风格与现有一致

验证：
- `cd frontend && npx tsc --noEmit` 无类型错误
- 组件可独立渲染 line/bar/pie 三种图表

完成标准：
- ChartView 导出为默认组件
- 支持 line/bar/pie 三种类型
- overrides prop 可覆盖 spec 字段
