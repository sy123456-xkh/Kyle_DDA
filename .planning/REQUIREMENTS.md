# Chat-to-BI MVP 重构需求

## 1. 项目范围

### 1.1 重构目标
全面提升代码质量、架构设计和功能完整性，将现有 MVP 从演示级别提升到可维护、可扩展的状态。

### 1.2 保留约束
- 目录结构：backend/ 和 frontend/ 分离
- 开发环境：macOS 本地运行
- 技术栈：Python/FastAPI + Next.js/TypeScript + DuckDB

### 1.3 不包含
- 生产环境部署
- 多租户/权限系统
- 企业级审计
- 性能优化（除非影响基本使用）

---

## 2. 后端重构需求

### 2.1 代码质量提升

**REQ-BE-001: 类型安全**
- 所有 API 端点使用 Pydantic 模型定义请求/响应
- 添加类型注解到所有函数
- 使用 mypy 进行类型检查

**REQ-BE-002: 错误处理**
- 统一的异常处理机制
- 用户友好的错误消息（中文）
- 结构化的错误响应格式
- 日志记录（结构化日志）

**REQ-BE-003: 代码规范**
- 遵循 PEP 8 规范
- 使用 black 格式化
- 添加必要的文档字符串

### 2.2 架构优化

**REQ-BE-004: 模块化设计**
- 服务层分离（dataset_service, query_service）
- 数据访问层抽象（repository pattern）
- 业务逻辑与路由分离

**REQ-BE-005: 依赖注入**
- DuckDB 连接池管理
- 配置管理（环境变量）
- 可测试的依赖注入

**REQ-BE-006: API 设计**
- RESTful 风格一致性
- 统一的响应格式
- API 版本控制（/api/v1）

### 2.3 安全加固

**REQ-BE-007: SQL 注入防护**
- 使用参数化查询
- SQL 语句白名单验证
- 增强的 SQL Guardrails

**REQ-BE-008: 文件上传安全**
- 文件类型验证（MIME type + 扩展名）
- 文件大小限制（可配置）
- 文件名清理（防止路径遍历）
- CSV 内容验证

**REQ-BE-009: 输入验证**
- dataset_id 格式验证
- 查询参数验证
- 请求体大小限制

### 2.4 数据库层改进

**REQ-BE-010: 连接管理**
- 连接池实现
- 事务管理
- 连接超时处理

**REQ-BE-011: 数据持久化**
- 数据集元数据存储
- 查询历史记录
- 数据清理策略（TTL）

## 9. 技术决策

### 9.1 状态管理方案
**选择：** React Context + useReducer
**理由：** 项目规模适中，避免引入额外依赖

### 9.2 API 客户端
**选择：** fetch + 自定义封装
**理由：** 原生支持，无需额外库

### 9.3 图表库
**选择：** Recharts 或 Chart.js
**理由：** React 友好，文档完善

### 9.4 测试框架
**选择：** pytest（后端）+ Vitest（前端）
**理由：** 生态成熟，易于集成
