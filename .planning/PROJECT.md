# Chat-to-BI MVP 全面重构

## 项目概述

对现有 Chat-to-BI 毕业项目进行全面重构，提升代码质量、架构设计和功能完整性，同时保持目录结构不变。

**当前状态：** Step 1-3 已完成（CSV上传、数据分析、基础查询、前端展示）

**重构目标：**
- 代码质量提升（类型安全、错误处理、规范化）
- 架构优化（模块化、可扩展性、可维护性）
- 安全加固（SQL注入防护、输入验证、文件上传安全）
- 功能完善（LLM集成、图表生成、高级查询）

## 技术栈

**后端：**
- Python 3.11 + FastAPI
- DuckDB（本地分析引擎）
- Pydantic（数据验证）

**前端：**
- Next.js 14 + TypeScript
- React 18
- Tailwind CSS

**开发环境：**
- macOS
- 本地开发运行

## 约束条件

### 必须保留
- 目录结构：backend/ 和 frontend/ 分离
- 开发环境：本地运行，不涉及部署

### 可以修改
- API 接口契约（路径、参数、响应格式）
- 前端 UI 布局
- DuckDB 数据模型
- 内部实现逻辑

### 不做的事
- 企业级审计
- 多租户计费
- 生产环境部署
- 复杂的权限系统

## 核心功能

1. **数据上传与分析**
   - CSV 文件上传
   - 自动字段类型推断
   - 数据质量分析（缺失率、样本值）

2. **自然语言查询**
   - 规则引擎（当前）
   - LLM 集成（待实现）
   - SQL 生成与执行

3. **结果展示**
   - 表格展示
   - 图表可视化（待实现）
   - SQL 查看与复制

4. **安全防护**
   - SQL 注入防护
   - 文件上传验证
   - 输入参数校验

## 成功标准

- 所有 API 端点有完整的类型定义和验证
- 前端组件模块化，状态管理清晰
- SQL 查询使用参数化，无注入风险
- 错误处理完善，用户友好的错误提示
- 代码符合 Python/TypeScript 最佳实践
- 核心功能有基础测试覆盖
